为什么用 Netty？

假设你要用 Java NIO(New IO) 实现服务器，要处理很多底层细节。

在这个例子中，我们用到了 Java NIO 的关键部分，如 Selector、ServerSocketChannel 和 SocketChannel，实现了一个简单的服务器，但也带来了很多手动处理的麻烦。

  1. 手动管理连接：代码需要自己设置 Selector 来监听通道的状态，并手动管理每个连接的生命周期。
  2. 多种事件处理：需要判断事件类型（如 OP_ACCEPT 和 OP_READ），并分别处理每种事件。
  3. 手动读写操作：用 ByteBuffer 手动管理数据的读取和写入。
  4. 资源管理：需要在每次事件完成后手动关闭通道，并手动清理缓冲区，以防止资源泄露。
  5. 异常处理：在实际的应用中，还需要加入更多的异常处理和错误恢复机制，否则服务器在遇到错误时可能崩溃。

如果使用 Netty，代码会简单很多

开发者可以集中精力在业务逻辑上。例如，在 Netty 中，你只需定义消息的处理逻辑，Netty 会帮你自动管理连接、数据读取/写入、并发处理等。

没有 Netty 时，需要手动处理：

  1. 连接和事件管理（选择器、注册事件、处理事件）。
  2. 数据的手动读取和缓冲区管理。
  3. 并发连接的管理。
  4. 错误处理和资源释放等。
